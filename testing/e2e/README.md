# Теоретический материал и инструкция по выполнению задания

## 1. Что такое сквозной сценарий (E2E-сценарий)

E2E (End-to-End) сценарий — это подробное описание пользовательского пути от начала до конца.  
Он показывает, как реальный пользователь выполняет полный процесс в приложении, включая все ключевые взаимодействия.

Примеры пользовательских потоков: авторизация, регистрация, покупка товара, публикация поста и т. д.

Цель E2E-сценария:
- проверить работу системы в целом;
- убедиться, что части приложения взаимосвязаны и работают корректно;
- увидеть реальное поведение системы с точки зрения пользователя.

### Структура E2E-сценария:
1. Наименование сценария  
2. Предусловия  
3. Последовательность шагов  
4. Ожидаемый результат  
5. Альтернативные пути (если есть)

### Пример E2E-сценария:
Наименование: Успешная авторизация  
Предусловия: пользователь зарегистрирован  
Шаги:
1. Открыть страницу входа  
2. Ввести корректный email  
3. Ввести корректный пароль  
4. Нажать кнопку "Войти"  

Ожидаемый результат: система авторизует пользователя и перенаправляет в личный кабинет.  
Альтернативный путь: введён неверный пароль — система показывает ошибку.

## 2. Что такое тест-кейс

Тест-кейс — формализованный документ, описывающий:
- цель теста,
- необходимые условия,
- действие по шагам,
- ожидаемый результат.

Тест-кейс должен позволять любому человеку воспроизвести тест без дополнительных объяснений.

### Структура тест-кейса:
1. ID  
2. Название  
3. Цель  
4. Предусловия  
5. Шаги выполнения  
6. Ожидаемый результат  
7. Постусловия (если есть)  
8. Примечания (если есть)

### Пример тест-кейса:
ID: TC-001  
Название: Успешная авторизация  
Цель: Проверить вход в систему с корректными данными  
Предусловия: пользователь зарегистрирован  

Шаги:
1. Открыть страницу входа  
2. Ввести email  
3. Ввести пароль  
4. Нажать кнопку "Войти"  

Ожидаемый результат: пользователь перенаправлен в личный кабинет.

## 3. Инструкция по выполнению задания

### Шаг 1. Выберите пользовательский поток
Выберите один вариант из списка предложенных сценариев (авторизация, регистрация и т. д.).

### Шаг 2. Составьте E2E-сценарий
Заполните следующие разделы:
- Наименование сценария  
- Предусловия  
- Последовательность шагов  
- Ожидаемый результат  
- Альтернативные пути  

При описании шагов важно указывать действия в строгой последовательности.

### Шаг 3. Выберите тест-кейс
Выберите один из предложенных вариантов тест-кейсов (например, "Поиск товара по ключевому слову").

### Шаг 4. Оформите тест-кейс
Заполните следующие пункты:
- ID  
- Название  
- Цель  
- Предусловия  
- Шаги выполнения  
- Ожидаемый результат  
- Постусловия  
- Примечания  

Шаги должны быть точными и однозначными.

### Шаг 5. Проверьте документ
Убедитесь, что:
- логика последовательная,
- шаги понятны,
- ожидаемый результат указан,
- структура полностью соблюдена.

### Шаг 6. Отправьте файл
Скопируйте шаблон задания, заполните его в Word или Google Docs и отправьте в общий чат.
